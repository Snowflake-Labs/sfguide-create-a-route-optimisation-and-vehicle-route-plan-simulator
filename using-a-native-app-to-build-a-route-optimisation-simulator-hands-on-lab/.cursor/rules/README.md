## Rules Conventions

- One rule per file, named by topic (e.g., `snowcli_test_connection.md`).
- Prefer imperative, step-by-step guides.
- Reference repo-relative paths in backticks.

Index:
- `overview.md` — Project overview and technologies
- `python_streamlit_standards.md` — Python/Streamlit coding standards
- `streamlit_deployment_standards.md` — Streamlit deployment template and rules
- `notebook_standards.md` — Authoring and deploying Snowflake notebooks (editable in Snowflake)
- `notebook_cell_metadata_standards.md` — Cell metadata (hidden/code/results) and safe cell operations
- `streamlit_comment_tagging.md` — Standard JSON comment for Streamlit apps
- `native_app_and_marketplace.md` — Native app and internal marketplace access
- `listings_workflow.md` — Creating synthetic data and organization listings
- `industry_customization_guide.md` — Building industry-specific scenarios
- `snowcli_test_connection.md` — Set up a SnowCLI test connection using `dataops/event/variables.yml`.
- `snowcli_test_connection_check.md` — Verify test connection exists via `snowcli_testing.yml` + helper script

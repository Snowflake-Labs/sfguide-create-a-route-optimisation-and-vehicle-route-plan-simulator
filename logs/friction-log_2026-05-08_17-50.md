## Friction Points

### 1. Workspace File Upload Challenge (HIGH)

**Problem:** `snow stage copy` and `PUT` commands are not available in the Snowflake Workspace web environment. Only `dbt`, `ls`, and `snow` (with limited subcommands) are supported.

**Attempted Solutions:**
- Tried `snow stage copy` → Not supported
- Tried `PUT file://...` via SQL → Unsupported format error
- Tried `COPY FILES` from workspace with nested paths → Created incorrect directory structure

**Final Solution:**
1. Created flat YAML files in workspace root (not nested in `temp_stage_upload/`)
2. Used `COPY FILES` from workspace stage URI: `snow://workspace/USER$.PUBLIC."sfguide-build-fleet-intelligence-with-cortex-code"/versions/live/`
3. Files landed at correct paths in `@OPENROUTESERVICE_APP.CORE.ORS_SPCS_STAGE/services/*/`

**Time Lost:** ~8 minutes (multiple upload attempts + cleanup)

**Recommendation:**
- Update SKILL.md to include explicit workspace deployment instructions
- Add example using `COPY FILES` for workspace environments
- Pre-stage YAML files at workspace root in the skill's `assets/` directory

## Summary of Learnings

1. **Workspace environments require specialized file upload patterns** - `COPY FILES` from workspace stage URI is the solution when `PUT`/`snow stage copy` are unavailable

2. **Pre-existing images saved ~15 minutes** - Building 5 container images from scratch would have added significant time

3. **Version validation is critical** - Image repository contents don't always match YAML specifications

4. **Modular deployment works** - Skipping optional modules (03 partial, 05, 06) didn't impact core functionality

5. **Compute pool startup is the main wait time** - ~3 minutes for pool to become ACTIVE, unavoidable

## Friction Points

### F1: DDL statements cannot be batched in snowflake_sql_execute

- **Step:** Step 3: Setup DB/stages
- **Severity:** Low
- **What happened:** The skill SKILL.md shows all DDL as a single code block (CREATE WAREHOUSE, CREATE DATABASE, CREATE SCHEMA x2, CREATE IMAGE REPOSITORY, CREATE STAGE x3), but the `snowflake_sql_execute` tool only supports one statement per call. Had to execute 8 separate tool calls.
- **Resolution:** Executed them sequentially, one per tool call. Also had to add `USE SCHEMA` before CREATE IMAGE REPOSITORY since it needed schema context.
- **Recommendation:** The skill's Step 3 SQL block should note that when using `snowflake_sql_execute`, each statement must be separate. Or provide an alternative single `snow sql -f` file for the infra setup. The IMAGE REPOSITORY creation also requires explicit schema context (`USE SCHEMA OPENROUTESERVICE_APP.CORE` or fully qualified name).

### F2: `npm run build` output invisible through tail pipe

- **Step:** Step 5: Build images (control app local build)
- **Severity:** Low
- **What happened:** Running `npm run build 2>&1 | tail -10` returned completely empty output even though the build succeeded (dist/ was created). The vite build output may use stderr or carriage returns that don't work well through tail.
- **Resolution:** Verified success by checking `ls dist/` after the command completed.
- **Recommendation:** In build-images.md, note that vite build output is unreliable through pipes. Suggest verifying via `ls -la dist/index.html` after build.

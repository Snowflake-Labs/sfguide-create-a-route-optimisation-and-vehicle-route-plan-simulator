## Friction Points

### F1. Service YAML image tags out-of-sync with `image-versions.env`
- **Symptom:** `01_core_infra.sql` failed creating `routing_gateway_service` and `ors_control_app` because their YAMLs referenced `routing_reverse_proxy:v1.0.5` and `ors_control_app:v1.0.214` while the env file (and the images we built) were `v1.0.0` and `v1.0.185`.
- **Resolution this run:** Edited `gateway/routing-gateway-service.yaml` and `ors_control_app/ors_control_app_service.yaml` to match `image-versions.env`, re-uploaded, re-ran modules (idempotent).
- **Recommendation:** Add a pre-flight check (already in skill text but not enforced) that diffs each `services/*/...yaml` image tag vs `image-versions.env` and refuses to upload if mismatch. Or consolidate: have the install script template tags into YAMLs from env at upload time.

### F2. Stage upload race vs `CREATE STAGE` (Step 7)
- **Symptom:** First `snow stage copy datasets/intro/...` failed with "Stage SEED_DATA_STAGE does not exist or not authorized" because the upload was kicked off in parallel with the stage creation.
- **Resolution:** Re-ran uploads after the CREATE STAGE call returned; all uploaded fine.
- **Recommendation:** Skill should call out clearly that the SEED_DATA_STAGE is created in Step 7 (not Step 3) and uploads must follow.

### F3. Retail-catchment seed: correlated subquery rejected
- **Symptom:** `INSERT INTO RETAIL_POIS ... AND COALESCE((SELECT ST_INTERSECTS ...), TRUE)` failed: `Unsupported subquery type cannot be evaluated`.
- **Resolution:** Commented out the polygon-refine subquery (the bbox prefilter is sufficient for the SF Bay Area). Final RETAIL_POIS=48,102, REGIONAL_ADDRESSES=2,827,959.
- **Recommendation:** Rewrite the polygon refine as a CROSS JOIN LATERAL or precompute the boundary into a temp table, then JOIN. Provide the rewritten SQL in `references/seed-data.sql` as the default.

### F4. Route-optimization seed: `ADD SEARCH OPTIMIZATION IF NOT EXISTS`
- **Symptom:** `ALTER TABLE ... ADD SEARCH OPTIMIZATION IF NOT EXISTS ON EQUALITY(ALTERNATE)` — `IF NOT EXISTS` not supported on this clause.
- **Resolution:** Removed `IF NOT EXISTS`.
- **Recommendation:** Update `references/seed-data.sql` accordingly. (Edit applied during this run.)

### F5. Route-optimization seed: `ARRAY_CONSTRUCT` in `VALUES` clause
- **Symptom:** `INSERT ... FROM VALUES ((..., ARRAY_CONSTRUCT(...), ...), ...)` — `Invalid expression [ARRAY_CONSTRUCT(...)] in VALUES clause`.
- **Resolution:** Refactored the LOOKUP seed to `SELECT ... ARRAY_CONSTRUCT(...) ... UNION ALL SELECT ...`.
- **Recommendation:** Update `references/seed-data.sql` to use the UNION ALL pattern. AGENTS.md already documents this Snowflake limitation under "Loading GEOGRAPHY Data".

### F6. Route-optimization stored proc short-circuits before LOOKUP/JOB_TEMPLATE
- **Symptom:** After F4/F5 fixes, `CALL SEED_ROUTE_OPTIMIZATION_REGION('SanFrancisco')` returned "Already seeded: 51146 places" because PLACES had been populated by the earlier failed run, so the proc skipped LOOKUP/JOB_TEMPLATE inserts. Result: PLACES=51146 but LOOKUP=0, JOB_TEMPLATE=0.
- **Resolution:** Manually `DELETE FROM PLACES WHERE REGION='SanFrancisco'` then re-called the proc. Final: PLACES=51146, LOOKUP=4, JOB_TEMPLATE=29.
- **Recommendation:** The procedure's "already seeded" early-return logic is too eager — it should check all three tables (or always populate LOOKUP/JOB_TEMPLATE since they're tiny and idempotent) before short-circuiting.

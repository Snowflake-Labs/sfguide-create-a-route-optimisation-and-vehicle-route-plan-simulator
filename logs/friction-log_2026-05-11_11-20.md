## Friction Points

### Friction #1 — SKILL.md Step 3 SQL syntax: `CREATE IMAGE_REPOSITORY` (with underscore)
**Location:** `.cortex/skills/build-routing-solution/SKILL.md` Step 3, line:
```
CREATE IMAGE_REPOSITORY IF NOT EXISTS IMAGE_REPOSITORY ...
```
**Issue:** Snowflake syntax is `CREATE IMAGE REPOSITORY` (two words, space) — not `CREATE IMAGE_REPOSITORY` (single token). Running the literal SQL block would fail with a syntax error if executed verbatim via `snowflake_sql_execute` or `snow sql -f`.
**Workaround applied this run:** Issued the corrected DDL `CREATE IMAGE REPOSITORY IF NOT EXISTS OPENROUTESERVICE_APP.CORE.IMAGE_REPOSITORY ...` which succeeded.
**Recommended fix:** Update SKILL.md Step 3 to read `CREATE IMAGE REPOSITORY IF NOT EXISTS IMAGE_REPOSITORY ...` (and ensure any helper scripts use the same).

### Friction #2 — `snow sql -f` output silent for retail-catchment / route-optimization seed scripts
**Location:** Step 8 demo deployment. Running `snow sql -f .../retail-catchment/references/seed-data.sql` and the equivalent for `route-optimization` produced no visible stdout in the captured terminal output (just blank lines), even after `tail -20`. Both ran for several minutes and ultimately succeeded (verified by querying row counts directly).
**Issue:** Without explicit progress indication, an operator can't tell whether the script is hung, network-stalled, or working. The bash background terminal showed only blank lines while the large Overture-Maps queries were executing.
**Workaround applied this run:** After waiting and seeing no output, queried `INFORMATION_SCHEMA.TABLES` directly to confirm rows were being populated. Both scripts ultimately completed successfully.
**Recommended fix:** Add a one-line "EXPECTED DURATION: 3-5 min — output may be silent during large CTAS over Overture Maps" note to the SKILL.md Quick Start sections of `retail-catchment` and `route-optimization`. Optionally, the seed scripts could `SELECT 'STEP X starting' AS msg;` markers between large CTAS statements so progress is visible in `snow sql -f` output.

### Friction #3 — route-optimization notebook deployment skipped

## Friction Points

### F1: REPO_URL JSON output line-wrap appearance

- **Step:** 5 — Get repository URL via `snow spcs image-repository url --format JSON`
- **Severity:** Low
- **What happened:** The terminal display rendered the JSON-extracted URL with what looked like an embedded newline (`pm-fleet-test...image_re\npository`). On inspection via `od -c` the underlying file was actually clean — only the terminal column-wrap caused the visual artifact.
- **Resolution:** Used `tr -d '\n'` defensively when reading the URL into subsequent build commands so any stray newline would be stripped before being appended to a docker tag.
- **Recommendation:** Update `references/build-images.md` Step 2 with a defensive one-liner:
  ```bash
  REPO_URL=$(snow spcs image-repository url ... --format JSON | python3 -c "import sys,json; print(json.load(sys.stdin)['message'].strip())" | tr -d '\n')
  ```
  This eliminates the perceived ambiguity and protects against future Snow CLI versions that might embed actual newlines.

### F2: Docker `--ignorefile` flag is Podman-only

- **Step:** 5 — Build `ors_control_app` image
- **Severity:** Low (already documented but easy to miss)
- **What happened:** The primary build-images.md command block uses `--ignorefile .dockerignore.prebuilt` which is unsupported by Docker. The reference does include a "Docker on ARM Mac" callout with the swap-file workaround, but it's separated from the main flow.
- **Resolution:** Followed the docker-specific block in `references/build-images.md`: `cp .dockerignore .dockerignore.bak && cp .dockerignore.prebuilt .dockerignore`, build, then `mv .dockerignore.bak .dockerignore`.
- **Recommendation:** Restructure `references/build-images.md` so the Docker and Podman ors_control_app flows are presented as a fork at the top of the section (e.g., "If $CONTAINER_CMD = docker → use swap; if podman → use --ignorefile"), instead of one being primary and the other a footnote. This avoids the agent reading the primary block first and copying an unsupported flag.

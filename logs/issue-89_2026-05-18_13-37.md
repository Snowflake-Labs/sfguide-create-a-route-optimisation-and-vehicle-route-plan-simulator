## Issues Encountered During Execution

### Issue 1: `routing` connection private key missing

- **Step:** Image registry login (Task 5)
- **Severity:** ERROR (recovered via workaround)
- **Category:** DEPLOYMENT_FAILURE / WORKAROUND

**What happened:**
The `routing` connection in `~/.snowflake/connections.toml` references `private_key_file: '/tmp/rsa_key.p8'`, but the file was missing. `snow spcs image-registry login -c routing` failed with `[Errno 2] No such file or directory: PosixPath('/private/tmp/rsa_key.p8')`. The `ukb96706` connection uses OAuth, which I expected to fail in non-interactive CLI mode — but `snow spcs image-registry login -c ukb96706` succeeded, presumably because of cached credentials.

**Resolution:**
Used `-c ukb96706` for all `snow spcs` and `snow stage` commands; OAuth had cached tokens that permitted non-interactive auth.

**Suggested fix:**
- Persist a documented procedure to regenerate `/tmp/rsa_key.p8` for the `routing` connection (it is in `/tmp/` so survives nothing across reboots).
- Or update `~/.snowflake/connections.toml` to point `routing` at a stable location like `~/.snowflake/keys/rsa_key.p8`.

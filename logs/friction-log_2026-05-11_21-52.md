## Friction Points

### 1. Session Context Switching (Low)
**What happened:** After creating SYNTHETIC_DATASETS database, subsequent commands targeting OPENROUTESERVICE_APP.CORE failed with "Schema 'SYNTHETIC_DATASETS.CORE' does not exist" because the session context auto-switched.  
**Resolution:** Added explicit `USE SCHEMA OPENROUTESERVICE_APP.CORE` before service creation.  
**Recommendation:** SKILL.md should note that CREATE DATABASE changes session context, or Module 01 should include USE SCHEMA after the database creation block.

### 2. Workspace-to-Stage File Paths (Medium)
**What happened:** COPY FILES from workspace preserves the full workspace-relative path on the target stage (e.g., `datasets/synthetic_ebikes/...` becomes the path on stage). Required understanding how the ORS config references files.  
**Resolution:** Used two-hop approach (workspace → temp stage → target stage with explicit paths).  
**Recommendation:** The upload-map-files subskill documents this well. The main SKILL.md could add a brief note about nested path handling for workspace deployments.

### 3. PROVISION_REGION Async Syntax (Low)
**What happened:** `EXECUTE IMMEDIATE ... ASYNC INTO` caused a SQL compilation error in the web environment.  
**Resolution:** Created a synchronous version of PROVISION_REGION that calls the wrapper directly.  
**Recommendation:** Document that ASYNC INTO requires specific Snowflake scripting support and provide a synchronous fallback.

### 4. Services Auto-Suspended During Data Loading (Low)
**What happened:** ORS_SERVICE, VROOM_SERVICE, and DOWNLOADER auto-suspended (300-600s timeout) while I was loading seed data.  
**Resolution:** Resumed all services after data loading completed.  
**Recommendation:** Consider setting AUTO_SUSPEND_SECS = 0 temporarily during initial deployment, then re-enabling after setup completes.
